Contexto

Se trata de un programa de gestión empresarial de un concesionario de coches de la marca Koenigsegg

Este programa es el programa de gestión del jefe del concesionario desde el cual podremos gestionar clientes, empleados y vehiculos

Datos tecnicos

Se ha usado una herencia de 2 clases padres llamadas Persona y Vehiculo:

  • Persona: 5 clases heredan de ella
  • Vehiculo: 1 clase hereda de ella
En cuanto a herramientas, se han usado:
  1. HashSet
  2. LinkedList
  3. List
  4. Map
  5. Scanner
  6. Set
  7. TreeMap

A la hora de ejecucion del programa, se instancian varios objetos de las clases anteriores para poder simular la existencia de clientes, gerentes, vendedores, jefe y varios vehiculos de distintos modelos con sus respectivos datos diferenciados los del mismo modelo por su color

El programa, antes de poder hacer nada, te obliga a escribir la contraseña de acceso (esto paras simular una pequeña seguridad basica en todos los sistemas de gestión). Si la contraseña es correcta, entonces ya podemos empezar a hacer las gestiones:

  1. Añadir clientes
  2. Gestionar empleados
  3. Gestionar coches
  4. Cambiar la contraseña

Añadir clientes

Esta opcion nos empieza a pedir datos (nombre, apellidos, DNI, etc) para poder almacenarlos en variables las cuales después seran usadas. Una vez todos los datos hayan sido rellenados, se empezara a crear el nuevo objeto de clientes dando de valores a sus atributos las variables que contienen los datos introducidos anteriormente por el usuario.

Gestionar empleados

  1. Gestionar vendedores: con esta subopcion tenemos otras 2 subopciones mas:
    • Añadir vendedores: esta opcion nos pedira otra vez datos a introducir por el usuario los cuales se almacenaran en variables para luego al crear el nuevo vendedor, estas variables definan los atributos del objeto
    • Eliminar vendedores: se introduce el nombre del vendedor a eliminar, se busca en la lista y si ese vendedor existe se elimina automaticamente del sistema
  2. Gestionar gerentes:
    • Añadir gerentes: mismo procedimiento como el de añadir vendedores
    • Eliminar gerentes: mismo procedimiento como el de eliminar vendedores pero en la lista de gerentes

Gestionar vehiculos

  1. Añadir coche: mismo procedimiento como el de añadir vendedores pero con bastantes mas datos al pedirse datos como el modelo, motor, ruedas, altura, anchura, etc
  2. Eliminar coche: mismo procedimiento como el de eliminar vendedores pero en la lista de coches

Cambiar contraseña

Se pedira la contraseña actual para hacer una verificacion, si la contraseña coincide entonces se pide al usuario introducir la nueva contraseña y asi cambiamos el valor de la contraseña
